# ME-Plan
## Q&A
### 背景意义类
#### 1 请描述一下国内外目前对于这个问题的研究进展。
当前一些复杂的规划任务，例如教学设计、心理干预和军事行动等，涉及多维约束、目标权衡等问题的解决。为此类任务生成高质量的解决方案，需要基于反馈进行迭代调整和优化。这已经成为现代问题解决框架的基石\
但该策略的有效性受到评估质量的制约，当前“LLM-as-a-Judge”范式通常采用静态和通用评分标准，可能存在评估短视问题，难以捕捉不同问题的潜在约束和领域特定的细微差别\
为了弥补这一差距，我们引入了ME-Plan，通过构建针对特定问题的认知全景图，实现对隐式约束的核心冲突的检测。
#### 2 你做研究的创新点在哪里？
（1）提出‘认知全景图’概念。 我们不是直接让LLM生成答案，而是先让元认知智能体挖掘任务的核心矛盾和隐式红线（比如‘不能说谎’），作为后续评估的‘宪法’。\
（2）动态评估标准生成。 传统方法用固定模板打分，而我们根据每个任务的认知全景图，从零合成专属评估标准，包含‘为什么这个维度重要’和‘具体怎么检查’。\
（3）细粒度内省审计。 评论者不直接给分，而是生成思维链轨迹，逐条指出哪里违规、为什么违规、怎么修改。这让评估过程可解释、可操作。

### 方法内容类
#### 1 请介绍一下整篇文章的研究思路。
认知全景图挖掘（Agent 1）：分析任务，输出核心矛盾 + 隐式红线。比如在谈判场景中，矛盾是‘想要高价 vs 不能说谎’，红线是‘不能做虚假陈述’。\
专属评估标准生成（Agent 3）：基于全景图，动态合成评估维度。每个维度包含Rationale（为什么重要） 和Method（怎么检查）。比如‘真实性’维度的Method是‘检查发言是否与已知事实矛盾’。\
细粒度内省审计（Agent 4）：不直接给分，而是生成思维链轨迹——逐条对照评估标准，指出方案的具体问题、违规依据、修改建议。\
反馈驱动优化：从思维链中提取可执行反馈，让Agent 2修订方案。实验表明通常一轮迭代即可达到最优。
#### 2 实证研究是如何进行的？
我们在Sotopia-hard基准上进行实验。这个基准包含20个高难度社交任务，涉及谈判、欺骗、说服等场景，每个任务都有冲突的目标和隐式约束。\
我们对比了三个基线：\
Simple CoT：标准思维链提示\
ToM-Agent：心智理论建模，预测对方心理状态\
SWM：社交世界模型，SWM采用S3AP形式主义将潜在的社会动态结构化为状态、观测和心智状态，从而增强了智能体表征和预测未来互动能力。\
评估采用7个维度：目标完成、可信度、关系、知识、保密、社会规范、财务收益。每个任务进行6次独立运行，评判模型温度设为0.0确保稳定性。\
结果显示，ME-Plan在综合得分上超越所有基线，比标准基线提升6.6%，尤其在保密（-0.03 vs -0.33）和社会规范（-0.08 vs -0.20）维度提升显著。

#### 3 论文中所用的数据是来自于哪里？
我们使用的是Sotopia-hard数据集，这是从SOTOPIA完整数据集中筛选出的20个最具挑战性的任务。\
SOTOPIA是由CMU提出的社交智能评估环境，它的构成是：\
90个社交场景：涵盖合作、竞争、谈判、欺骗等\
40个角色：每个角色有年龄、职业、性格、秘密等详细设定\
5类关系：家人、朋友、恋人、熟人、陌生人\
450个任务组合：场景×角色×关系的交叉\
Sotopia-hard是从这450个任务中选出的20个最难任务，用于测试模型在冲突场景下的表现。每个任务包含共享场景描述和各自的私有目标，智能体需要在不知道对方目标的情况下，通过对话达成自己的目标。

### 展望未来类
#### 1 研究不足有哪些？
第一，推理延迟较高。 ME-Plan需要四个智能体协作和多轮迭代，比单次生成慢，不适合实时应用场景。\
第二，依赖基础模型能力。 我们的上限受限于底层LLM的知识和推理能力。如果模型本身不知道某个隐式规则，认知全景图也无法挖掘出来。\
第三，领域验证有限。 目前主要在社交交互场景验证，在纯逻辑任务（如数学证明、代码生成）上的效果还需要进一步验证。

### 其他问题
#### 1 研究过程中遇到过什么问题，怎么解决的？
我们在实现认知全景图自动提取时遇到过问题。初始提示词效果不好，模型提取的约束要么太泛化（比如‘遵守法律’），要么与任务无关。\

我们通过迭代优化提示词解决：先人工标注几个高质量示例，明确‘核心矛盾’要指出两个对立需求，‘隐式红线’要是具体的、可检查的约束。然后在提示词中加入few-shot示例，逐步优化。这个过程我参与了部分标注工作。\

另外，我们也遇到过LLM输出格式不遵守JSON Schema的问题。解决方案是在提示词中先给出格式示例，然后在代码中加入重试机制——如果解析失败，重新调用LLM并要求修正格式。通常重试1-2次就能成功。
